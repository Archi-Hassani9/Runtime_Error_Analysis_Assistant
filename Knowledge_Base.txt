Question: What is a runtime error?
Answer: A runtime error is an error that occurs while a program is executing. It happens after the program has started running and may cause it to crash or behave unexpectedly.

Question: When do runtime errors occur?
Answer: Runtime errors occur during the execution phase of a program, after successful compilation and when the program is running.

Question: What is the difference between syntax error and runtime error?
Answer: A syntax error occurs due to incorrect code structure and prevents compilation, while a runtime error occurs during execution after the program starts running.

Question: What is the difference between compile-time error and runtime error?
Answer: Compile-time errors occur during code compilation and prevent execution, whereas runtime errors occur during execution after compilation succeeds.

Question: Why do runtime errors occur?
Answer: Runtime errors occur due to invalid operations such as division by zero, accessing invalid memory, incorrect input, or logic errors in the program.

Question: What happens when a runtime error occurs?
Answer: When a runtime error occurs, the program may crash, terminate abruptly, or produce incorrect results depending on the type of error.

Question: Can runtime errors be detected before execution?
Answer: No, runtime errors cannot be detected before execution because they occur only when the program is running.

Question: Are runtime errors always fatal?
Answer: No, some runtime errors can be handled using exception handling mechanisms, allowing the program to continue execution.

Question: What are common examples of runtime errors?
Answer: Common examples include division by zero, null pointer access, file not found, and array index out of bounds.

Question: What is an exception in runtime errors?
Answer: An exception is a specific type of runtime error that can be caught and handled using programming constructs like try-catch blocks.

Question: How are runtime errors different from logical errors?
Answer: Runtime errors cause the program to crash or stop, while logical errors allow the program to run but produce incorrect results.
